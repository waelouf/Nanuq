## 1. Console.dev — Tool Directory Submission

Submit via console.dev/tools (tool listing form, NOT newsletter pitch)

```
Tool name: Nanuq
URL: https://github.com/waelouf/Nanuq
One-line description: Unified open-source web UI for Kafka, Redis, RabbitMQ, AWS SQS, and Azure Service Bus
Category: Developer Tools / Infrastructure

Description:
Nanuq is a self-hostable web UI that replaces 5 separate browser tabs for
messaging and caching system management. One Docker Compose command runs the
full stack.

Supports:
- Kafka: topic browser, consumer group offset monitoring, message publish
- Redis: all 6 data types (Strings, Hashes, Lists, Sets, Sorted Sets, Streams)
- RabbitMQ: exchanges, queues, bindings, message send/receive
- AWS SQS/SNS: queue and topic management across 15 regions
- Azure Service Bus: queues, topics, subscriptions across 30+ regions

AES-256 encrypted credential storage. MIT licensed. .NET 9 backend, Vue.js 3
frontend, 478 automated tests (0 failures).

GitHub: github.com/waelouf/Nanuq
License: MIT
```

**When to submit:** Anytime — tool directory, not gated on a release event. Submit
this on the same day as the Dev.to post for maximum signal.
