## LinkedIn

### Post 1 — Developer problem angle (post on Dev.to publish day)

```
I got tired of managing 5 messaging systems from 5 different browser tabs.

Kafka UI for topics. RedisInsight for keys. RabbitMQ management console for queues. AWS console for SQS. Azure portal for Service Bus. Every debugging session started with a 20-minute tab-cycling warmup.

The commercial tools that unify this cost hundreds to thousands per month. The open-source alternatives each covered one system well.

So I built Nanuq: a single Docker Compose UI that manages all five.

Kafka, Redis, RabbitMQ, AWS SQS/SNS, Azure Service Bus — credential management, CRUD operations, message browsing and publishing. AES-256 encrypted credential storage. MIT licensed.

Just published a write-up on Dev.to covering the architecture and the test coverage sprint (5% to 81% in two weeks). Link in comments.

github.com/waelouf/Nanuq
```
